# Mid-Term Reflection

## Materi yang Paling Sulit

Materi yang paling sulit saya kuasai selama Minggu 1–7 adalah memahami arsitektur MVC (Model, View, Controller), terutama dalam membedakan tanggung jawab masing-masing bagian.

- Model digunakan untuk mengelola data dan logika database.
- View digunakan untuk tampilan antarmuka pengguna.
- Controller digunakan sebagai penghubung antara Model dan View.

Awalnya saya sering bingung menentukan kode harus diletakkan di folder controller, model, atau views. Selain itu, memahami alur data dari user hingga ke database juga cukup membingungkan.

## Cara Mengatasinya

Saya mengatasinya dengan:
- mencoba membuat project MVC sederhana secara langsung,
- mempelajari struktur folder seperti `controllers`, `models`, `views`, dan `docs`,
- membaca ulang materi serta melihat contoh implementasi,
- dan rutin melakukan latihan menggunakan Git dan GitHub agar lebih terbiasa dengan workflow development.

Dengan latihan bertahap, saya mulai memahami fungsi masing-masing komponen dan bagaimana MVC membantu membuat kode lebih rapi dan mudah dikelola dalam kerja tim.
